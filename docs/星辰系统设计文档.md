## 一、流程

### 1. 销售订单

#### 1.1 录入流程

销售在 CRM 做前置工作（付款方式审批、特殊价格审批等），确定订单后，在 ERP 录入订单。

**录入方式：**
- 在页面单个创建
- 表格批量导入
- 接口传输，按单号查询填充

**录入内容：**
- **基础信息**
  - 订单号
  - 订单名称
  - 客户
  - 客户名称
  - 联系人
  - 联系人电话
  - 联系人邮箱
  - 客户地址
  - 抵运国
  - 外销币种
- **产品明细**
- **费用**

#### 1.2 审核流程

```
销售发起审核
  └─ 【商务】审核
       ├─ 审核内容：待补充
       ├─ 审核通过 ──> 【采购】审核
       │                 ├─ 审核内容：待补充
       │                 ├─ 审核通过 ──> 【采购助理】确认生成发货需求
       │                 └─ 驳回 ──> 驳回，销售修改
       └─ 驳回 ──> 驳回，销售修改
```

#### 1.3 订单状态

| 状态 | 说明 |
|------|------|
| 待提交 | — |
| 审批中 | — |
| 已驳回 | — |
| 备货中 | — |
| 备货完成 | — |
| 部分发运 | 待定 |
| 已发货 | — |
| 已作废 | — |

### 3. 收款单

#### 3.1 流程

```
财务创建收款单（可能拆单）
  └─ 发布招领
       └─ 销售认领（仅能全额认领）
            ├─ 销售确认 ──> 已确认，认领完成
            └─ 销售可能认领错误
                 └─ 财务协助退回至待确认状态
                      └─ 销售重新认领并确认 ──> 已确认，认领完成
```

#### 3.2 状态

- 登记
- 待认领
- 待确认
- 已确认
- 已作废

#### 3.3 操作

- 新建
- 收款调整
- 回退

### 5. 发货需求

#### 5.1 流程

```
【采购助理】确认生成发货需求
  └─ 采购补充信息
       └─ 仓库补充信息
            └─ 商务安排发运
```

#### 5.2 状态

| 状态 | 说明 |
|------|------|
| 待处理 | 未确认走库存还是走采购 |
| 采购中 | 需要走采购的部分 |
| 已备齐库存 | 采购已经到货，或者发现有的备货库存 |
| 待发运 | 库存都已经备齐，生成物流单了，但还未发运 |
| 已发运 | 生成了物流单，且已经发运 |

#### 1.4 收款单状态 & 操作

**登记**
- **说明：** 财务新建的收款单，还未发布，还不能认领
- **支持操作：**
  - 编辑：财务操作，编辑收款信息部分
  - 发布：财务发布收款单，发布后变为待认领状态
  - 作废：财务操作，作废后变为已作废状态

**待认领**
- **说明：** 财务已经发布招领的收款单，等待销售认领
- **支持操作：**
  - 认领：销售操作，填写认领信息保存后，变为待确认状态
  - 作废：财务操作，作废后变为已作废状态

**待确认**
- **说明：** 销售已经填写了认领信息，但是销售还未确认，此时允许销售修改
- **支持操作：**
  - 修改认领：销售操作，支持修改认领信息，保存后状态不变更
  - 确认认领：销售操作，确认认领后状态变更为已确认
  - 取消认领：销售操作，取消认领后，清空认领信息，状态变更为待认领
  - 收款调整：财务操作，支持修改收款信息，保存后状态不变更

**已确认**
- **说明：** 销售已经操作确认的收款单，表示已经认领完成。注意：只有确认完才是有效的认领，认领的金额才会关联到订单上
- **支持操作：**
  - 转待确认：财务操作，将收款单转到待确认状态，让销售可以修改
  - 收款调整：财务操作，支持修改收款信息，保存后将状态变更为待确认，待销售再次确认

**已作废**
- **说明：** 财务作废的收款单
- **支持操作：**
  - 删除：财务操作，将已作废的收款单删除

#### 1.5 操作汇总

| 操作 | 操作人 | 限制条件 | 说明 |
|------|--------|----------|------|
| 新建 | 财务 | 任何状态 | — |
| 编辑 | 财务 | 仅【登记】状态 | — |
| 发布 | 财务 | — | 发布招领 |
| 认领 | 销售 | — | 一个收款单仅能认领一次 |
| 确认 | 销售 | — | 认领后做确认动作，确认后销售不能再修改 |
| 转登记 | 财务 | 仅【待认领】状态 | 财务填写错误需要修改（貌似不需要，删了重建就行？） |
| 取消认领 | 销售 | 仅【待确认】状态 | 取消后收款单回到待认领状态，其他人可重新认领 |
| 转待确认 | 财务 | 【已确认】状态 | 让销售可以修改认领信息 |
| 收款调整 | 财务 | 【待确认、已确认】 | 财务发现填写错了需要修改；方案1：线下沟通直接调整；方案2：调整后回到待确认状态让销售再确认 |
| 作废 | 财务 | 仅【登记、待认领】状态 | — |
| 删除 | 财务 | 仅【作废】状态 | — |

#### 3.3 销售订单状态 & 操作

**待提交**
- **支持操作：**
  - 编辑：销售操作，支持编辑订单
  - 提交审核：销售操作，提交后状态变更为审核中
  - 作废：销售操作，作废后状态变更为已作废
  - 修改额外查看人：销售操作，可以增加人或者减少人

**审核中**
- **支持操作：**
  - 修改额外查看人：销售操作，可以增加人或者减少人

**审核通过**
- **支持操作：**
  - 生成发货需求：销售操作，生成后订单状态变更为备货中
  - 转待提交：销售操作，可以将订单转回待提交状态
  - 修改收发信息：销售操作，支持修改收发相关联系信息
    - 基础信息：起运地、目的地
    - 收发通信息：收货人公司、收货人其他信息、通知人公司、通知人其他信息、发货人
    - 内销收货信息：客户公司、客户收货信息
  - 修改额外查看人：销售操作，可以增加人或者减少人

**已驳回**
- **支持操作：**
  - 编辑：销售操作，支持编辑订单
  - 提交审核：销售操作，提交后状态变更为审核中
  - 作废：销售操作，作废后状态变更为已作废
  - 修改额外查看人：销售操作，可以增加人或者减少人

**备货中**
- **支持操作：**
  - 修改发运信息：销售操作，若已生成物流单则不允许修改
    - 基础信息：起运地、目的地
    - 收发通信息：收货人公司、收货人其他信息、通知人公司、通知人其他信息、发货人
    - 内销收货信息：客户公司、客户收货信息
  - 修改额外查看人：销售操作，可以增加人或者减少人

**备货完成**
- **支持操作：**
  - 修改发运信息：销售操作，若已生成物流单则不允许修改
    - 基础信息：起运地、目的地
    - 收发通信息：收货人公司、收货人其他信息、通知人公司、通知人其他信息、发货人
    - 内销收货信息：客户公司、客户收货信息
  - 修改额外查看人：销售操作，可以增加人或者减少人

**部分发货**
- **支持操作：**
  - 修改额外查看人：销售操作，可以增加人或者减少人

**已发货**
- **支持操作：**
  - 修改额外查看人：销售操作，可以增加人或者减少人

**已作废**
- **支持操作：**
  - 删除：销售操作，支持删除已作废的订单

#### 3.4 操作汇总

| 操作 | 操作人 | 限制条件 | 说明 |
|------|--------|----------|------|
| 新建 | 销售 | 任何状态 | — |
| 编辑 | 销售 | 【待提交、已驳回】 | 审批通过后按需使用单独修改功能 |
| 提交审核 | 销售 | — | 发起审核 |

#### 5.3 发货需求状态 & 操作

**待分配库存**
- **支持操作：**
  - 分配库存：采购操作，基于产品明细操作，在弹窗中标记是否需要采购，填写采购数量和关联现有库存批次；已操作过的支持再次操作（相当于修改）
  - 取消分配库存：采购操作，基于产品明细操作，取消后清空【是否需要采购、需采购数量】，并将关联批次库存解绑
  - 生成采购申请：采购操作，基于整个发货需求操作，一个发货需求只能有一个采购申请；需要校验所有产品明细的【是否需要采购】是否都有值
  - 确认备货完成：采购操作，当整个发货需求全部使用现有库存时展示，基于整个发货需求操作
  - 作废：采购助理操作，基于整个发货需求操作；作废后清空【是否需要采购、需采购数量】并解绑批次库存，同时将上游单据的【是否生成发货需求】改为否，回退上游单据状态
  - 更新到货信息：采购助理操作，基于产品明细操作，填写【预计工厂发货时间、实际工厂发货时间、实际到货时间】，支持多次修改，每次覆盖更新
  - 标记配件是否单独开票：采购操作，基于产品明细操作，仅配件支持标记

**采购中**
- **支持操作：**
  - 更新到货信息：采购助理操作，基于产品明细操作，填写【预计工厂发货时间、实际工厂发货时间、实际到货时间】，支持多次修改，每次覆盖更新
  - 标记配件是否单独开票：采购操作，基于产品明细操作，仅配件支持标记

**已备货完成**
- **支持操作：**
  - 修改库存分配：采购操作，只能修改批次库存或者仓库，不能再回到采购流程；保存时校验各产品数量与发货需求数量是否相等
  - 更新到货信息：采购助理操作，填写【预计工厂发货时间、实际工厂发货时间、实际到货时间】，支持多次修改，每次覆盖更新
  - 标记配件是否单独开票：采购操作，基于产品明细操作，仅配件支持标记

**待发货**
- **支持操作：** 无

**已发货**
- **支持操作：** 无

**已作废**
- **支持操作：** 无
