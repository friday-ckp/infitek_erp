# Implementation Readiness Assessment Report

**Date:** 2026-04-15
**Project:** infitek_erp

## PRD 分析

### 功能需求（FR）

#### 用户认证（FR-A 系列）

- **FR-A01**：系统支持多用户账号管理，内置初始管理员账号（用户名 + 密码），管理员可创建其他用户账号；MVP 阶段所有账号具有相同全系统操作权限，无角色分配
- **FR-A02**：用户账号密码以不可逆加密方式存储，不存储明文；用户可修改自己密码；管理员可重置其他账号密码
- **FR-A03**：管理员可在设置页面创建、查看、编辑、停用用户账号；停用账号立即失效，持有 Token 下次请求时被拒绝；所有业务操作自动记录操作账号（created_by / updated_by）

#### 主数据 — 产品体系（FR01–FR06）

- **FR01**：产品专员可创建、查看列表、查看详情、编辑三级产品分类（最多三级层级）
- **FR02**：产品专员可在产品分类下创建、查看列表、查看详情、编辑 SPU（品名、品牌、基础描述）
- **FR03**：产品专员可在 SPU 下创建、查看列表、查看详情、编辑多个 SKU 变体，含规格、重量、体积、HS 码、报关品名（中英文）、申报价值参考
- **FR04**：产品专员可为产品创建和维护 FAQ 条目（问题 + 回答）
- **FR05**：产品专员可在产品证书库中创建、查看列表、查看详情、编辑证书记录，含证书类型（CE/FCC 等）、有效期、发证机构、适用 SPU 关联
- **FR06**：产品专员可在产品资料库中创建、查看列表、查看详情、编辑资料记录（说明书、规格书等），含文件上传和关联 SPU/SKU

#### 主数据 — 供应链参考数据（FR07–FR11）

- **FR07**：采购员可创建、查看列表、查看详情、编辑供应商档案，含公司名称、联系人、账期（天数）、付款方式、开户行信息
- **FR08**：采购员可为供应商关联一个或多个采购品类
- **FR09**：授权用户可创建、查看列表、查看详情、编辑合同条款范本；状态含草稿 / 已审批两个值；已审批状态的条款方可被采购订单引用
- **FR10**：授权用户可创建、查看列表、查看详情、编辑港口信息（港口名称、国家、港口代码）
- **FR11**：授权用户可创建、查看列表、查看详情、编辑物流供应商档案（名称、联系方式、服务类型）

#### 主数据 — 客户与公司主体（FR12–FR13）

- **FR12**：授权用户可创建、查看列表、查看详情、编辑客户主数据，含公司名称、联系信息、开票需求、客户所在国家
- **FR13**：授权用户可创建、查看列表、查看详情、编辑公司主体信息，含纳税人识别号、海关备案号、检疫备案号、开户行/银行账号/SWIFT CODE、默认币种、签订地点

#### 主数据 — 基础参考数据（FR14–FR17）

- **FR14**：系统管理员可管理单位信息（新增、编辑、启用/禁用）
- **FR15**：系统管理员可管理仓库信息（名称、地址、启用/禁用）
- **FR16**：系统管理员可管理币种信息（币种代码、名称、启用/禁用）
- **FR17**：系统管理员可管理国家/地区参考数据（名称、代码）

#### 搜索与查询（FR18–FR20）

- **FR18**：授权用户可在任意主数据模块内按关键字搜索记录（至少匹配名称字段）
- **FR19**：授权用户可通过多个条件组合筛选主数据列表
- **FR20**：所有列表页支持分页加载

#### 销售订单管理（FR21–FR25）

- **FR21**：授权用户可通过表单创建销售订单（草稿不入库），填写完整后提交直接创建为"已确认"状态；需关联已有客户，添加一条或多条订单行（关联 SKU、数量、单价），选择币种；须选择订单类型（普通/售后/样品），MVP 仅启用"普通订单"
- **FR22**：销售订单通过表单提交直接创建为"已确认"状态，不支持撤销至草稿
- **FR23**：授权用户可查看销售订单列表（支持按状态、客户筛选）和订单详情
- **FR24**：销售订单确认后，授权用户可在详情页点击"生成发货需求"按钮手动触发生成对应发货需求单；发货需求包含订单中所有 SKU 及应发数量；生成时系统自动查询每个 SKU 在各仓库的当前可用库存
- **FR25**：系统在关联发货出库单全部确认发货后，自动将销售订单状态更新为"已发货"

#### 发货需求与物流（FR26–FR34）

- **FR26**：发货需求详情展示每个 SKU 行的应发数量及各仓库当前可用库存；每个 SKU 行含履行类型字段（全部采购 / 部分采购 / 使用现有库存）；选择"使用现有库存"或"部分采购"时展开内联仓库分配区，用户填写使用数量后系统按 FIFO 自动完成库存锁定
- **FR27**：授权用户可从发货需求单创建采购订单（请购型）；一个发货需求可创建多张采购订单（通常按供应商分组）；请购型收货后自动触发库存锁定
- **FR28**：授权用户可从发货需求单创建物流单
- **FR29**：授权用户可查看发货需求单列表（支持按状态、关联销售订单筛选）和详情
- **FR30**：发货需求单状态随关联出库确认自动更新：全部 SKU 出库完成后状态变为"已发运"
- **FR31**：授权用户可创建物流单，关联发货需求单；字段分组：基本信息（物流供应商、运输方式、公司主体、起运港、目的港/目的地、运抵国）、物流跟踪（ETD/ETA/订舱号等，确认后补填）、装箱信息（SKU/箱数/每箱数量/尺寸/毛重）、报关信息（是否出口报关/唛头）、备注
- **FR32**：授权用户通过提交物流单表单完成创建（草稿不入库），系统直接生成"已确认"状态物流单；物流跟踪字段可在确认后于详情页随时补填或更新
- **FR33**：授权用户可从物流单创建发货出库单
- **FR34**：授权用户可查看物流单列表（支持按状态筛选）和详情

#### 采购订单管理（FR35–FR38）

- **FR35**：授权用户可通过表单创建采购订单（草稿不入库），填写完整后提交直接创建为"已确认"状态；可从发货需求单预填 SKU 和数量（请购型）；关联已有供应商，补充单价；合同条款默认关联供应商下状态为"已审批"的第一条范本；须在创建时标注"请购型"或"备货型"
- **FR36**：采购订单通过表单提交直接创建为"已确认"状态，不支持撤销至草稿
- **FR37**：授权用户可查看采购订单列表（支持按状态、供应商筛选）和订单详情
- **FR38**：系统在关联收货入库单确认收货后，自动将采购订单状态更新为"已收货"

#### 库存管理（FR39–FR49）

- **FR39**：系统以双层结构维护库存：汇总层（SKU + 仓库）含实际库存、锁定量、可用库存（= 实际库存 - 锁定量）；批次层（SKU + 仓库 + 入库批次）含入库日期、来源单据、该批次实际数量、该批次锁定量；两层数据须保持一致
- **FR40**：系统管理员可为指定 SKU 和仓库录入或覆盖期初库存数量；期初录入确认后在批次层创建来源标注为"期初录入"的批次记录；支持重复提交以覆盖已有数据；库存数量仅通过收货入库和发货出库操作变动
- **FR41**：系统在发货出库操作确认后，实时扣减对应 SKU 在对应仓库的实际库存，并释放对应锁定量
- **FR42**：系统自动记录每次库存变动流水（变动类型、SKU、仓库、数量变化、操作人、操作时间）；变动类型：期初录入、采购入库、发货出库、锁定、解锁
- **FR43**：授权用户可按 SKU 或仓库查询当前库存数量（实际/锁定/可用）及历史变动流水
- **FR44**：授权用户可创建收货入库单，关联已确认的采购订单，逐行填写每个 SKU 的实际收货数量和目标仓库；收货确认后在批次层创建对应批次记录
- **FR45**：确认收货后系统自动触发：① 批次层创建新批次记录并更新汇总层实际库存；② 请购型按 FIFO 自动跨批次将入库数量锁定至对应发货需求；备货型仅增加实际库存，不触发锁定；③ 采购订单状态更新为"已收货"
- **FR46**：授权用户可从物流单创建发货出库单，逐行填写每个 SKU 的实际发货数量（允许部分发货）和来源仓库；库存扣减以实际发货数量为准
- **FR47**：系统在创建发货出库单时校验出库数量不超过该仓库中该 SKU 被本发货需求锁定的数量；超出时阻止提交并提示当前本需求锁定数量
- **FR48**：确认发货后系统自动触发对应 SKU 在来源仓库的实际库存扣减，并释放对应的锁定量
- **FR49**：授权用户可对状态为"待分配库存"（初始状态，尚未创建任何关联采购订单或物流单）的发货需求执行作废操作；处于"处理中"及之后状态的发货需求不允许作废；作废需二次确认；作废确认后系统触发：① 解锁所有锁定库存；② 关联销售订单状态回退为"已确认"；③ 销售订单详情页展示"重新生成发货需求"按钮；作废后状态变为"已作废"，不可再操作

**功能需求总计：52 条**（FR-A01~A03 共 3 条 + FR01~FR49 共 49 条）

### 非功能需求（NFR）

#### 性能（NFR-P 系列）

- **NFR-P1**：主数据列表页首屏加载时间 < 2 秒（标准网络，100 条以内数据）
- **NFR-P2**：主数据详情页加载时间 < 1.5 秒
- **NFR-P3**：搜索和筛选操作响应时间 < 1 秒
- **NFR-P4**：销售订单、采购订单提交（状态变更）响应时间 < 2 秒
- **NFR-P5**：收货入库、发货出库确认操作（含库存更新）响应时间 < 3 秒
- **NFR-P6**：库存查询（当前库存数量）响应时间 < 1 秒

#### 安全（NFR-S 系列）

- **NFR-S1**：所有数据传输使用 HTTPS 加密
- **NFR-S2**：用户密码以不可逆加密方式存储，不存储明文
- **NFR-S3**：用户连续无操作超过 8 小时后自动登出（时长可配置）
- **NFR-S4**：所有接口须通过身份验证，系统拒绝未授权请求

#### 可靠性（NFR-R 系列）

- **NFR-R1**：系统目标可用性 99%（允许计划内维护窗口）
- **NFR-R2**：数据每日自动备份，支持按需恢复

#### 可用性体验（NFR-U 系列）

- **NFR-U1**：系统面向桌面浏览器优先，支持 Chrome / Edge 最新两个主版本
- **NFR-U2**：所有用户操作的错误提示须清晰说明出错原因，不暴露技术错误码或堆栈信息
- **NFR-U3**：发货出库库存不足的校验提示须明确显示当前可用库存数量

#### 文件存储（NFR-F 系列）

- **NFR-F1**：产品资料库和产品证书库的文件上传及读取使用阿里云 OSS；服务端不在本地磁盘持久化文件内容
- **NFR-F2**：文件删除执行物理删除（从 OSS 永久删除），不保留软删除记录；删除操作需二次确认

**非功能需求总计：17 条**（P6 + S4 + R2 + U3 + F2）

### 补充约束与假设

- **单租户系统**：星辰科技内部使用，无多租户隔离需求，无订阅计划或计费模块
- **MVP 权限模型**：不实现 RBAC，所有登录用户拥有全系统操作权限；Post-MVP 补充自定义角色 + 模块级权限
- **多币种**：销售订单支持外销币种（USD、EUR 等），MVP 阶段不做汇率自动换算
- **集成**：MVP 阶段无外部系统集成，系统独立运行
- **浏览器优先**：Web 浏览器（桌面），移动端不在 MVP 范围
- **审计字段预留**：MVP 不要求变更审计日志，但数据模型应预留 created_at、updated_at、created_by、updated_by 字段
- **草稿不入库**：销售订单、采购订单、物流单的填写过程不产生数据库记录，提交后直接创建为已确认状态

### 对齐问题

#### 🚨 严重问题（阻塞实现）

**问题 1：缺少文件上传/存储后端架构定义**

#### ⚠️ 中等问题（需解决）

**问题 3：axios 拦截器无法按严重度路由到 Message vs. Notification**

- 架构：所有错误通过 axios 拦截器统一调用 `antdMessage.error()`（自动消失）
- UX-FB-2/FB-3：重要业务规则失败（如库存不足）应使用 `Notification.error`（持久可关闭，含单据链接）
- 当前单一拦截器无法区分"普通失败"（Message）与"重要业务失败"（Notification）

**问题 4：URL 筛选条件持久化无实现规范**

- UX-NAV-1：列表页筛选条件存入 URL query 参数，浏览器后退可恢复
- 架构文档未提及此需求，未定义实现方式（如 `useSearchParams` 钩子或自定义方案）

**问题 5：乐观更新模式无具体实现规范**

- UX-FB-1 要求：UI 先更新，后端确认，失败回滚
- 架构提到 TanStack Query 支持乐观更新，但未提供标准模式、代码示例或规则，开发者实现可能不一致

**问题 6：键盘快捷键管理无架构定义**

- UX-A11Y-5：Ctrl+K 全局搜索、Ctrl+S 保存表单（拦截浏览器默认行为）
- 架构未提及全局键盘事件管理、快捷键注册机制，或 Ctrl+S 与 Ant Design Form 提交的交互方式

---

#### 📋 低优先级问题

**问题 7：自定义组件目录结构未定义**

- UX 规范定义 7 个自定义组件（FlowProgress、SmartButton、InventoryIndicator、PurchaseGroupPreview、StatCard、ActivityTimeline、EntitySearchSelect）
- 架构仅命名 `components/common/`（通用 CRUD 组件）和 `components/order/OrderStatusTimeline`
- 7 个自定义组件无对应目录，且 `OrderStatusTimeline` 与 UX 规范中的 `ActivityTimeline` 命名不一致

**问题 8：前端性能目标缺失**

**问题 9：滚动位置恢复无策略**

- UX 规范：从详情页跳转关联单据后，浏览器后退键返回原页面应保持滚动位置
- React Router v7 默认不恢复滚动位置，架构未提及解决方案

**问题 10：防抖工具标准缺失**

- UX-EI-4 规定搜索输入框 300ms 防抖
- 架构未定义防抖工具（无共享 `useDebounce` 钩子，无 lodash.debounce 方案）

**问题 11：axe-core 无障碍 CI 集成**

- UX-A11Y-8 要求将 axe-core 集成到 CI/CD，每次构建自动扫描 WCAG AA 违规
- 架构将 CI/CD 明确推迟到 Post-MVP；前端测试仅要求关键表单组件快照测试，与此需求直接冲突
