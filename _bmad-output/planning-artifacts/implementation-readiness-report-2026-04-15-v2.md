#### 9. 库存管理

| ID | 需求描述 |
|----|---------|
| FR39 | 系统以双层结构维护库存：汇总层（SKU+仓库，含实际/锁定/可用三字段）+ 批次层（SKU+仓库+批次，含入库日期/来源单据/批次数量/批次锁定量）；两层须保持一致 |
| FR40 | 系统管理员可为指定 SKU 和仓库录入或覆盖期初库存；期初录入创建来源为"期初录入"的批次记录；支持重复提交覆盖；不提供日常库存调整功能 |
| FR41 | 发货出库确认后，系统实时扣减实际库存并释放锁定量 |
| FR42 | 系统自动记录每次库存变动流水（变动类型、SKU、仓库、数量变化、操作人、操作时间）；变动类型包括：期初录入、采购入库、发货出库、锁定、解锁 |
| FR43 | 授权用户可按 SKU 或仓库查询当前库存数量（实际/锁定/可用）及历史变动流水 |
| FR44 | 授权用户可创建收货入库单，关联已确认采购订单，逐行填写 SKU 实际收货数量和目标仓库；确认后在批次层创建对应批次记录 |
| FR45 | 收货确认后系统自动触发：① 创建批次记录并更新汇总层实际库存；② 请购型 PO 按 FIFO 自动跨批次锁定入库数量至对应发货需求，备货型 PO 仅增加实际库存不锁定；③ 采购订单状态更新为"已收货" |
| FR46 | 授权用户可从物流单创建发货出库单，逐行填写 SKU 实际发货数量（允许部分发货）和来源仓库；库存扣减以实际发货数量为准 |
| FR47 | 创建发货出库单时，系统校验出库数量不超过该 SKU 在该仓库被本发货需求锁定的数量；超出则阻止提交并提示当前锁定数量 |
| FR48 | 发货出库确认后，系统自动扣减对应 SKU 在来源仓库的实际库存并释放对应锁定量 |
| FR49 | 授权用户可对"待分配库存"状态的发货需求执行作废操作（处理中及之后状态不允许）；作废需二次确认；确认后：① 释放所有锁定库存；② 关联销售订单状态回退为"已确认"；③ 销售订单显示"重新生成发货需求"按钮；作废后状态变为"已作废"，不可再操作 |

### 额外约束与说明

- **单租户**：内部单租户系统，无多租户隔离需求
- **权限 MVP**：MVP 阶段无 RBAC，所有登录用户有全系统操作权限
- **集成 MVP**：MVP 无外部系统集成
- **多币种**：支持多币种字段，但 MVP 不做汇率自动换算
- **状态机**：
  - 销售订单：已确认 → 已发货
  - 采购订单：已确认 → 已收货
  - 发货需求：待分配库存 → 处理中 → 待发运 → 已发运 / 已作废
  - 物流单：已确认 → 已出库
- **审计预留**：数据模型预留 created_at、updated_at、created_by、updated_by 字段
- **组件复用**：15 类主数据模块应抽象标准化 CRUD 组件
