# Implementation Readiness Assessment Report

**Date:** 2026-04-15
**Project:** infitek_erp

## PRD 分析

### 功能需求（Functional Requirements）

**用户认证（FR-A 系列）**

- FR-A01：系统支持多用户账号管理；初始管理员账号（用户名+密码），管理员可创建其他用户账号；MVP 阶段所有账号全系统操作权限
- FR-A02：用户密码以不可逆加密存储，不存储明文；用户可修改自己密码；管理员可重置他人密码
- FR-A03：管理员可创建/查看/编辑/停用用户账号；停用账号立即失效，Token 在下次请求时被拒绝；所有操作自动记录操作账号（created_by / updated_by）

**主数据 -- 产品体系（FR01-FR06）**

- FR01：产品专员可创建/查看列表/查看详情/编辑三级产品分类（最多三级）
- FR02：产品专员可在产品分类下创建/查看列表/查看详情/编辑 SPU（品名、品牌、基础描述）
- FR03：产品专员可在 SPU 下创建/查看列表/查看详情/编辑多个 SKU 变体，含规格、重量、体积、HS 码、报关品名（中英文）、申报价值参考
- FR04：产品专员可为产品创建和维护 FAQ 条目（问题+回答）
- FR05：产品专员可在产品证书库中创建/查看/编辑证书记录，含证书类型、有效期、发证机构、适用 SPU 关联
- FR06：产品专员可在产品资料库中创建/查看/编辑资料记录（说明书、规格书等），含文件上传和关联 SPU/SKU

**主数据 -- 供应链参考数据（FR07-FR11）**

- FR07：采购员可创建/查看/编辑供应商档案，含公司名称、联系人、账期、付款方式、开户行信息
- FR08：采购员可为供应商关联一个或多个采购品类
- FR09：授权用户可创建/查看/编辑合同条款范本（条款名称+内容文本）；状态含草稿/已审批；已审批状态方可被采购订单引用
- FR10：授权用户可创建/查看/编辑港口信息（港口名称、国家、港口代码）
- FR11：授权用户可创建/查看/编辑物流供应商档案（名称、联系方式、服务类型）

**主数据 -- 客户与公司主体（FR12-FR13）**

- FR12：授权用户可创建/查看/编辑客户主数据，含公司名称、联系信息、开票需求、所在国家
- FR13：授权用户可创建/查看/编辑公司主体信息，含纳税人识别号、海关备案号、检疫备案号、开户行/账号/SWIFT CODE、默认币种、签订地点

**主数据 -- 基础参考数据（FR14-FR17）**

- FR14：系统管理员可管理单位信息（新增、编辑、启用/禁用）
- FR15：系统管理员可管理仓库信息（名称、地址、启用/禁用）
- FR16：系统管理员可管理币种信息（币种代码、名称、启用/禁用）
- FR17：系统管理员可管理国家/地区参考数据（名称、代码）

**搜索与查询（FR18-FR20）**

- FR18：授权用户可在任意主数据模块内按关键字搜索记录（至少匹配名称字段）
- FR19：授权用户可通过多个条件组合筛选主数据列表
- FR20：所有列表页支持分页加载

**销售订单管理（FR21-FR25）**

- FR21：授权用户可通过表单创建销售订单（草稿不入库），提交后直接创建为"已确认"状态；关联客户、订单行（SKU+数量+单价）、订单币种；须选择订单类型（普通/售后/样品），MVP 仅启用普通订单
- FR22：销售订单通过表单提交直接创建为"已确认"，不支持撤销至草稿
- FR23：授权用户可查看销售订单列表（支持按状态、客户筛选）和详情
- FR24：系统在销售订单确认时自动生成对应发货需求单（含所有 SKU 及应发数量），并自动查询每个 SKU 各仓库可用库存展示于发货需求详情
- FR25：系统在关联发货出库单全部确认发货后，自动将销售订单状态更新为"已发货"

**发货需求与物流（FR26-FR34）**

- FR26：发货需求详情展示每个 SKU 应发数量及各仓库可用库存；每个 SKU 行含履行类型（全部采购/部分采购/使用现有库存）；选择使用库存时展开内联仓库分配区，展示批次明细，用户填写使用数量确认后系统按 FIFO 自动跨批次完成库存锁定
- FR27：授权用户可从发货需求单创建采购订单（可多张，按供应商分组），SKU 行仅含全部采购或部分采购的条目，数量自动预填；此类采购订单为请购型
- FR28：授权用户可从发货需求单创建物流单
- FR29：授权用户可查看发货需求单列表（支持按状态、关联销售订单筛选）和详情
- FR30：发货需求单状态随关联出库确认自动更新；全部 SKU 出库完成后变为"已发运"
- FR31：授权用户可创建物流单，关联发货需求单；字段分组：基本信息（必填）、物流跟踪（确认后补填）、装箱信息（子行项，必填）、报关信息（选填）、备注
- FR32：物流单通过提交表单创建（草稿不入库），直接生成"已确认"状态；物流跟踪字段在确认后可在详情页随时补填/更新
- FR33：授权用户可从物流单创建发货出库单
- FR34：授权用户可查看物流单列表（支持按状态筛选）和详情

**采购订单管理（FR35-FR38）**

- FR35：授权用户可通过表单创建采购订单（草稿不入库），可从发货需求预填 SKU 和数量；关联供应商；合同条款默认关联该供应商已审批的第一条范本；须标注类型（请购型/备货型）
- FR36：采购订单通过表单提交直接创建为"已确认"，不支持撤销至草稿
- FR37：授权用户可查看采购订单列表（支持按状态、供应商筛选）和详情
- FR38：系统在关联收货入库单确认收货后，自动将采购订单状态更新为"已收货"

**库存管理（FR39-FR49）**

- FR39：系统以双层结构维护库存：汇总层（SKU+仓库）含实际库存/锁定量/可用库存；批次层（SKU+仓库+入库批次）含入库日期/来源单据/批次实际数量/批次锁定量；两层须保持一致
- FR40：系统管理员可为指定 SKU 和仓库录入或覆盖期初库存；期初确认后在批次层创建来源标注为"期初录入"的批次记录；支持重复提交覆盖
- FR41：系统在发货出库操作确认后，实时扣减对应 SKU 在对应仓库的实际库存，并释放对应锁定量
- FR42：系统自动记录每次库存变动流水（变动类型、SKU、仓库、数量变化、操作人、操作时间）；变动类型：期初录入/采购入库/发货出库/锁定/解锁
- FR43：授权用户可按 SKU 或仓库查询当前库存数量（实际/锁定/可用）及历史变动流水
- FR44：授权用户可创建收货入库单，关联已确认的采购订单，逐行填写每个 SKU 的实际收货数量和目标仓库
- FR45：确认收货后系统自动触发：① 批次层创建新批次并更新汇总层实际库存；② 根据采购订单类型处理锁定（请购型按 FIFO 自动锁定至对应发货需求，备货型仅增加实际库存）；③ 采购订单状态更新为"已收货"
- FR46：授权用户可从物流单创建发货出库单，逐行填写每个 SKU 的实际发货数量（允许部分发货）和来源仓库
- FR47：系统在创建发货出库单时校验出库数量不超过该仓库中该 SKU 被本发货需求锁定的数量；超出则阻止提交并提示
- FR48：授权用户确认发货后，系统自动触发对应 SKU 在来源仓库的实际库存扣减，并释放对应锁定量
- FR49：授权用户可对状态为"待分配库存"的发货需求执行作废操作；需二次确认；作废后自动解锁所有锁定库存、关联销售订单状态回退为"已确认"、展示"重新生成发货需求"按钮；作废不可撤销

**FR 总数：52 条（FR-A01~A03，FR01~FR49）**

### 非功能需求（Non-Functional Requirements）

**性能（Performance）**

- NFR-P1：主数据列表页首屏加载 < 2 秒（标准网络，100 条以内数据）
- NFR-P2：主数据详情页加载 < 1.5 秒
- NFR-P3：搜索和筛选操作响应 < 1 秒
- NFR-P4：销售订单、采购订单提交（状态变更）响应 < 2 秒
- NFR-P5：收货入库、发货出库确认操作（含库存更新）响应 < 3 秒
- NFR-P6：库存查询（当前库存数量）响应 < 1 秒

**安全（Security）**

- NFR-S1：所有数据传输使用 HTTPS 加密
- NFR-S2：用户密码以不可逆加密方式存储，不存储明文
- NFR-S3：用户连续无操作超过 8 小时后自动登出（时长可配置）
- NFR-S4：所有接口须通过身份验证，系统拒绝未授权请求

**可靠性（Reliability）**

- NFR-R1：系统目标可用性 99%（允许计划内维护窗口）
- NFR-R2：数据每日自动备份，支持按需恢复

**可用性体验（Usability）**

- NFR-U1：系统面向桌面浏览器优先，支持 Chrome / Edge 最新两个主版本
- NFR-U2：所有用户操作的错误提示须清晰说明出错原因，不暴露技术错误码或堆栈信息
- NFR-U3：发货出库库存不足的校验提示须明确显示当前可用库存数量

**文件存储（File Storage）**

- NFR-F1：产品资料库和产品证书库的文件上传及读取使用阿里云 OSS；服务端不在本地磁盘持久化文件
- NFR-F2：文件删除执行物理删除（从 OSS 永久删除），不保留软删除记录；删除操作需二次确认

**NFR 总数：17 条**
