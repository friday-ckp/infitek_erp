### 附加约束与假设

- **单租户系统**：单一组织、单一数据空间、无多租户隔离需求
- **MVP 无 RBAC**：所有登录用户有全系统操作权限；Post-MVP 补充角色权限
- **MVP 无外部集成**：无 CRM/海关数据库集成
- **多币种支持（简版）**：MVP 阶段不做汇率自动换算
- **审计字段预留**：数据模型预留 created_at/updated_at/created_by/updated_by，不做 UI 展示
- **Web 浏览器优先**：桌面端，移动端不在 MVP 范围

### 架构质量评估

#### 优势

1. **技术栈选型成熟**：NestJS 11 + TypeORM + MySQL 8 + React 19 + Ant Design 5 均为社区主流稳定版本，版本组合无已知冲突
2. **库存并发方案明确**：`QueryRunner + SELECT FOR UPDATE + 指数退避重试（最多 3 次）`，是最高风险点的最完整方案
3. **类型共享策略**：`packages/shared` 共享枚举/DTO，彻底消除前后端类型漂移，是本架构最大的一致性保障
4. **AI Agent 执行规范**：10 条强制规则 + 4 条反模式示例，可直接指导 Story 实现，极少模糊地带
