## 销售订单（SalesOrder）

```mermaid
stateDiagram-v2
    [*] --> 待提交: 创建订单

    待提交 --> 审核中: 提交审核
    待提交 --> 已作废: 作废

    审核中 --> 审核通过: 审核通过
    审核中 --> 已驳回: 驳回
    审核中 --> 已作废: 作废

    已驳回 --> 审核中: 重新提交
    已驳回 --> 已作废: 作废

    审核通过 --> 备货中: 手动触发生成发货需求
    审核通过 --> 已作废: 作废

    备货中 --> 备货完成: 所有 SKU 库存锁定完成
    备货完成 --> 部分发货: 部分出库单确认
    部分发货 --> 已发货: 全部出库单确认（自动）
    备货完成 --> 已发货: 全部出库单确认（自动）

    已作废: 已作废（终态）
    已发货: 已发货（终态）
```

## 发货需求（ShippingDemand）

> 说明：
> - 同一发货需求内，不同 SKU 明细行（甚至同一 SKU）可能同时存在"需要采购"和"使用现有库存"两种履行方式。
> - 状态只向前推进，不可逆。
> - 集中确认后，若所有 SKU 均使用现有库存，直接跳过"采购中"进入"已备货完成"。
> - 集中确认后，若有任一 SKU 需采购，先进入"待生成采购单"；采购订单创建成功后才进入"采购中"。
> - 每批请购型采购到货后 FIFO 自动锁定，直到所有应发数量全部锁定到位，推进至"已备货完成"。
> - 物流单只能使用已锁定的库存数量创建，因此出库数量与采购订单是否全部完成无关，状态由出库完成度决定。
> - 英文状态链：`pending_allocation` -> `pending_purchase_order` -> `purchasing` -> `prepared`。

```mermaid
stateDiagram-v2
    [*] --> 待分配库存: 销售订单审核通过后手动触发生成

    待分配库存 --> 已备货完成: 集中确认-全部SKU使用现有库存-库存锁定完成
    待分配库存 --> 待生成采购单: 集中确认-含至少一行需采购的SKU
    待分配库存 --> 已作废: 作废（仅此状态可作废）

    待生成采购单 --> 采购中: 请购型采购订单创建成功
    采购中 --> 已备货完成: 请购型收货后自动锁定-所有应发数量均已锁定到位

    已备货完成 --> 部分发货: 部分出库单确认
    部分发货 --> 已发货: 全部出库单确认（自动）
    已备货完成 --> 已发货: 全部出库单确认（自动）

    已作废: 已作废（终态）自动释放锁定库存-销售订单回退至审核通过
    已发货: 已发货（终态）
```

## 采购订单（PurchaseOrder）

```mermaid
stateDiagram-v2
    [*] --> 待确认: 创建采购订单

    待确认 --> 待供应商确认: 内部确认
    待供应商确认 --> 待收货: 供应商确认
    待收货 --> 部分收货: 部分收货入库单确认
    部分收货 --> 全部收货: 全部收货入库单确认（自动）
    待收货 --> 全部收货: 全部收货入库单确认（自动）

    待确认 --> 已取消: 取消
    待供应商确认 --> 已取消: 取消

    全部收货: 全部收货（终态）
    已取消: 已取消（终态）
```

> 取消说明：采购订单取消后，发货需求状态**不回退**，仍停在"采购中"。对应 SKU 的缺口由用户从**发货需求详情页**复用"创建采购订单"入口补单，系统每次收货后继续触发 FIFO 锁定检查，直到全部锁定到位才推进至"已备货完成"。

## 物流单（LogisticsOrder）

```mermaid
stateDiagram-v2
    [*] --> 已确认: 创建即确认（草稿不入库）物流跟踪字段确认后可随时补填

    已确认 --> 已发运: 关联出库单全部确认后自动触发
    已发运 --> 已送达: 手动更新
    已确认 --> 已取消: 取消

    已送达: 已送达（终态）
    已取消: 已取消（终态）
```

## 发货出库单（OutboundOrder）

> 出库单无独立状态机，创建即为终态，确认后自动触发库存扣减和上游状态联动。
