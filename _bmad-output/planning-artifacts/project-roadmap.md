# infitek_erp 项目路线图

## 核心原则

1. **每个交付阶段对应一份独立 PRD** -- 不做"全量 PRD"
2. **每份 PRD 独立走完 BMAD 流程**：PRD → 架构(CA) → 史诗/故事(CE) → 冲刺计划(SP) → 开发(DS)
3. **前一期的产出是后一期的输入** -- 数据模型和 API 接口向后兼容
4. **产品简报覆盖整体愿景**，路线图管阶段划分，PRD 管具体需求

## 阶段划分

### 第一期：主数据层（当前阶段）

**PRD 文件**：`prd-phase1-master-data.md`
**目标**：建立统一、可信的数据基础，为所有交易模块提供单一数据来源

**范围 -- 纳入的模块/实体：**

| 来源模块 | 纳入实体 | 说明 |
|---------|---------|------|
| 基础数据 | 单位信息、仓库管理、币种信息管理、国家/地区管理 | 全部 4 个表单 |
| 产品管理 | 产品分类管理、SPU信息管理、产品SKU管理、产品资料库、产品证书库、FAQ管理 | 全部 6 个表单 |
| 采购管理 | 采购供应商管理、合同条款范本 | 仅主数据部分 |
| 商务管理 | 港口信息、物流供应商 | 仅主数据部分 |
| 财务管理 | 公司主体信息 | 仅主数据部分 |
| 销售管理 | 客户管理 | 仅主数据部分 |

**关键交付物：**
- 15 类主数据实体的 CRUD + 导入/导出
- 历史数据迁移模板（从宜搭/Excel 迁入）
- 产品证书有效期预警机制

**明确排除：** 所有交易单据、审批流、外部系统集成

---

### 第二期：销售管理 + 采购管理

**PRD 文件**：`prd-phase2-sales-procurement.md`
**目标**：打通订单到采购的核心链路

**范围：**

| 模块 | 纳入实体 | 说明 |
|------|---------|------|
| 销售管理 | 销售订单管理 | 订单录入（单建/批导）、审批流、状态追踪 |
| 采购管理 | 采购价格管理、采购申请单管理、采购订单管理、PO合同审批 | 按单触发采购、PO审批 |
| 财务管理 | 销售价格管理、待维护销售价格管理 | 价格体系支撑销售 |
| 外部集成 | CRM（富通天下）数据直连 | 定时拉取销售订单至 ERP，减少手工录入 |

**前置依赖：** 一期主数据层全部上线（产品SKU、供应商档案、客户档案）

---

### 第三期：库存管理 + 商务管理

**PRD 文件**：`prd-phase3-inventory-logistics.md`
**目标**：仓储履行与发货跟单

**范围：**

| 模块 | 纳入实体 | 说明 |
|------|---------|------|
| 库存管理 | 收货通知单、质检整改单、发货通知单、入库单、批次库存记录表、出库单 | 全部 6 个表单 |
| 商务管理 | 发货需求管理、物流单管理、物流费用管理 | 核心跟单流程 |
| 商务管理 | 打印发货需求委托书、打印报关单、打印清关单 | 单据打印 |
| 采购管理 | 分配库存（发货需求） | 库存分配到发货需求 |

**前置依赖：** 二期销售订单 + 采购订单上线

---

### 第四期：财务管理

**PRD 文件**：`prd-phase4-finance.md`
**目标**：应收应付，多币种对账

**范围：**

| 模块 | 纳入实体 | 说明 |
|------|---------|------|
| 财务管理 | 资金账户管理、收款登记与认领 | 多对多收款认领、付款审批 |

**前置依赖：** 三期库存/商务上线（完整订单-发货-收款链路）

---

### 第五期：外部集成 + 高级功能

**PRD 文件**：`prd-phase5-integration.md`
**目标**：系统间打通，消除最后的手工断点

**范围：**
- 合思费控付款审批对接
- 售后管理系统退换货追踪
- 物流全链路追踪
- BI 看板 / 管理驾驶舱

**前置依赖：** 四期财务管理上线

## PRD 命名规范

```
prd-phase{N}-{英文简称}.md
```

示例：
- `prd-phase1-master-data.md`
- `prd-phase2-sales-procurement.md`
- `prd-phase3-inventory-logistics.md`
- `prd-phase4-finance.md`
- `prd-phase5-integration.md`

## 每份 PRD 的 BMAD 工作流

```
[CP] 创建 PRD
  → [VP] 验证 PRD（可选）
  → [EP] 编辑 PRD（如有修改）
  → [CU] 创建 UX 设计（如有 UI）
  → [CA] 创建架构设计
  → [CE] 创建史诗和故事
  → [IR] 检查实施就绪性
  → [SP] 冲刺计划
  → [CS/DS/CR] 故事开发循环
  → [ER] 回顾（可选）
```

每个阶段独立走完这个流程。启动新阶段的 PRD 时，**先读本路线图**确认范围。

## 跨阶段注意事项

- **数据模型向后兼容**：一期定义的主数据表结构，后续阶段只增不改（除非通过 [CC] 变更流程）
- **接口预留**：每期架构设计时需考虑下一期的集成点，但不实现
- **共享实体归属**：如果一个实体（如"客户管理"）的主数据在一期建立，其交易功能在二期实现，PRD 中要明确标注依赖关系

## 变更管理

如果需要调整阶段划分或模块归属：
1. 使用 `[CC] Correct Course` 流程
2. 更新本路线图
3. 受影响的 PRD 通过 `[EP] Edit PRD` 同步更新
