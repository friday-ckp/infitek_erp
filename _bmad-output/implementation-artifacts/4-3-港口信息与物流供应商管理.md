# Story 4.3: 港口信息与物流供应商管理

Status: done

## Story

As a 授权用户,
I want 管理港口信息（港口类型、港口/机场代码、中英文名称、国家/地区）和物流供应商档案（名称、联系电话、地址、邮箱、合作状态、等级、简称、国家/地区、自动编码、联系人、默认合作主体）,
So that 创建物流单时可以直接选择港口和物流供应商，不用手动输入。

---

## Acceptance Criteria

**Given** 授权用户进入港口信息列表页
**When** 页面加载完成
**Then** 展示港口列表（ProTable），含列：港口名称、国家/地区、港口/机场代码
**And** 支持按国家筛选、关键字搜索、分页

**Given** 授权用户点击"新建港口"
**When** 填写港口类型（必填，下拉单选：起运港/目的港）、港口/机场代码（必填）、港口中文名（必填）、港口英文名（必填）、国家/地区（下拉单选，关联国家/地区主数据）
**Then** 港口记录创建成功

**Given** 授权用户进入物流供应商列表页
**When** 页面加载完成
**Then** 展示物流供应商列表（ProTable），含列：供应商名称、联系电话、联系人、合作状态、国家/地区、供应商等级、创建时间
**And** 支持关键字搜索、分页

**Given** 授权用户点击"新建物流供应商"
**When** 填写供应商名称（必填）、联系电话（必填）、公司详细地址（必填）、联系邮箱（必填）、合作状态（下拉单选：合作/淘汰）、供应商等级（5 星选择）、供应商简称（必填）、国家/地区（下拉单选）、联系人（必填）、默认合作主体（下拉单选，关联公司主体）
**And** 供应商编码由系统自动生成，规则为 `YCWL + 4位数自增`
**Then** 物流供应商记录创建成功

**Given** 授权用户在港口或物流供应商详情页
**When** 点击"编辑"
**Then** 可修改所有字段并保存

## Dev Agent Context

### 字段参考（Step 2.a）

参考 [docs/系统模块对应字段清单.md](/Users/chenkangping/.codex/worktrees/ce00/infitek_erp/docs/系统模块对应字段清单.md):

- 港口信息字段：港口类型（起运港/目的港）、港口/机场代码、港口中文名、港口英文名、国家/地区
- 物流供应商字段：供应商名称、联系电话、公司详细地址、联系邮箱、合作状态（合作/淘汰）、供应商等级（1-5）、供应商简称、国家/地区、供应商编码（自动生成：`YCWL0001`）、联系人、默认合作主体

本 Story 的最低交付必须满足 AC 中的必填 CRUD 字段；其余参考字段可一并落地，前提是不引入额外业务流程耦合。

### 关键实现约束

1. 所有 Entity 必须继承 `BaseEntity`，并给每个持久化字段添加 `@Expose()` 与显式 `@Column({ name })`
2. `ports` / `logistics-providers` 采用软删除，与 `countries` / `warehouses` 保持一致
3. 所有枚举必须定义在 `packages/shared`，禁止前后端各自维护
4. 端口国家、物流商国家使用软引用 `countryId` + 冗余 `countryName`
5. 物流供应商不包含“服务类型”字段；供应商编码必须在后端自动生成，前端只读展示
6. 列表页必须沿用 `SearchForm + ProTable + TanStack Query + useDebouncedValue` 模式
7. 表单页禁止直接调用 axios；国家选项查询复用 `/countries`
8. 面包屑与侧边栏必须同步更新，否则用户无法正常导航

### 枚举建议

export const PortType = {

  DESTINATION: '目的港',
} as const;

} as const;
