# Story 1.3: 用户认证（登录、JWT 守卫、Token 过期）

Status: ready-for-dev

## Story

As a 系统用户,
I want 通过用户名和密码登录系统，获取 JWT Token 访问受保护的接口,
So that 我的操作有身份标识，系统拒绝未授权访问。

## Acceptance Criteria

**AC1 — 登录成功返回 JWT Token**
- Given 系统已初始化内置管理员账号（用户名 admin）
- When 用正确的用户名和密码调用 `POST /api/v1/auth/login`
- Then 返回 JWT Access Token（有效期 8 小时）
- And Token payload 包含用户 ID 和用户名
- And 响应格式遵循统一格式 `{ success: true, data: { accessToken: "..." }, message: "OK", code: "OK" }`

**AC2 — JWT 守卫保护受保护接口**
- Given 用户持有有效 JWT Token
- When 调用任意受保护接口并在 Authorization Header 携带 `Bearer <token>`
- Then 请求正常通过，接口返回预期数据

**AC3 — 未授权请求被拒绝**
- Given 用户未携带 Token 或 Token 无效
- When 调用任意受保护接口（非 `/api/v1/auth/login`）
- Then 返回 401 Unauthorized 错误
- And 响应格式遵循统一错误格式 `{ success: false, data: null, message: "...", code: "UNAUTHORIZED" }`

**AC4 — Token 过期被拒绝**
- Given 用户 Token 已超过 8 小时
- When 使用过期 Token 调用接口
- Then 返回 401 错误，提示 Token 已过期（满足 NFR-S3）

**AC5 — 密码 bcrypt 加密存储**
- Given 数据库中 users 表已创建
- When 查看 admin 用户记录
- Then 密码字段为 bcrypt 哈希值（rounds=12），不存储明文（满足 NFR-S2）

**AC6 — 登录失败返回业务错误**
- Given 用户名或密码错误
- When 调用 `POST /api/v1/auth/login`
- Then 返回 401 错误，message 为"用户名或密码错误"
- And 不暴露是用户名错误还是密码错误（防止用户枚举攻击）

**AC7 — 停用账号无法登录**
- Given 某用户账号已被停用（status = 'inactive'）
- When 该用户尝试登录
- Then 返回 401 错误，message 为"账号已停用"

**AC8 — 全局 JWT 守卫注册**
- Given `AppModule` 已配置
- When 查看全局守卫注册
- Then `JwtAuthGuard` 已通过 `APP_GUARD` 全局注册
- And `/auth/login` 通过 `@Public()` 装饰器豁免鉴权
- And 健康检查 `/health` 通过 `@Public()` 装饰器豁免鉴权

### JWT Strategy 实现参考

```typescript
// apps/api/src/modules/auth/strategies/jwt.strategy.ts
import { Injectable } from '@nestjs/common';
import { PassportStrategy } from '@nestjs/passport';
import { ExtractJwt, Strategy } from 'passport-jwt';
import { ConfigService } from '@nestjs/config';

@Injectable()
export class JwtStrategy extends PassportStrategy(Strategy) {
  constructor(configService: ConfigService) {
    super({
      jwtFromRequest: ExtractJwt.fromAuthHeaderAsBearerToken(),
      ignoreExpiration: false,
      secretOrKey: configService.get<string>('JWT_SECRET'),
    });
  }

  async validate(payload: { sub: number; username: string }) {
    return { id: payload.sub, username: payload.username };
  }
}
```

### AuthService 实现参考

```typescript
// apps/api/src/modules/auth/auth.service.ts
import { Injectable, UnauthorizedException } from '@nestjs/common';
import { JwtService } from '@nestjs/jwt';
import { UsersService } from '../users/users.service';
import * as bcrypt from 'bcrypt';
import { UserStatus } from '../users/entities/user.entity';

@Injectable()
export class AuthService {
  constructor(
    private usersService: UsersService,
    private jwtService: JwtService,
  ) {}

  async login(username: string, password: string) {
    const user = await this.usersService.findByUsername(username);

    // 用户不存在或密码错误：统一返回相同错误，防止用户枚举
    if (!user || !(await bcrypt.compare(password, user.password))) {
      throw new UnauthorizedException({ code: 'INVALID_CREDENTIALS', message: '用户名或密码错误' });
    }

    if (user.status === UserStatus.INACTIVE) {
      throw new UnauthorizedException({ code: 'ACCOUNT_DISABLED', message: '账号已停用' });
    }

    const payload = { sub: user.id, username: user.username };
    return { accessToken: this.jwtService.sign(payload) };
  }
}
```

### JwtAuthGuard 实现参考

```typescript
// apps/api/src/common/guards/jwt-auth.guard.ts
import { ExecutionContext, Injectable, UnauthorizedException } from '@nestjs/common';
import { Reflector } from '@nestjs/core';
import { AuthGuard } from '@nestjs/passport';
import { IS_PUBLIC_KEY } from '../decorators/public.decorator';
import { JsonWebTokenError, TokenExpiredError } from 'jsonwebtoken';

@Injectable()
export class JwtAuthGuard extends AuthGuard('jwt') {
  constructor(private reflector: Reflector) {
    super();
  }

  canActivate(context: ExecutionContext) {
    const isPublic = this.reflector.getAllAndOverride<boolean>(IS_PUBLIC_KEY, [
      context.getHandler(),
      context.getClass(),
    ]);
    if (isPublic) return true;
    return super.canActivate(context);
  }

  handleRequest(err: any, user: any, info: any) {
    if (info instanceof TokenExpiredError) {
      throw new UnauthorizedException({ code: 'TOKEN_EXPIRED', message: 'Token 已过期，请重新登录' });
    }
    if (err || !user) {
      throw new UnauthorizedException({ code: 'UNAUTHORIZED', message: '未授权访问' });
    }
    return user;
  }
}
```

### 注意事项

1. **UserStatus 枚举位置**：`UserStatus` 枚举定义在 `user.entity.ts` 中（仅后端使用），不需要放入 `packages/shared`。若 Story 1.4 前端需要展示用户状态，届时再迁移到 shared。

2. **@Exclude() 与 ClassSerializerInterceptor**：`password` 字段标注 `@Exclude()`，配合 Story 1.2 已全局注册的 `ClassSerializerInterceptor`，序列化时自动排除密码字段。**必须同时在 Entity 上标注 `@Expose()` 才能让其他字段正常序列化**（ClassSerializerInterceptor 默认排除所有未标注 @Expose 的字段）。

3. **JWT_SECRET 环境变量**：`.env.example` 已有 `JWT_SECRET` 和 `JWT_EXPIRES_IN=8h`，开发时需在 `.env` 中设置真实值。

4. **Migration 执行**：本 Story 完成后，开发者需手动执行 `pnpm --filter api typeorm migration:run` 创建 users 表并插入 admin 账号。

5. **健康检查豁免**：`HealthController` 需添加 `@Public()` 装饰器，否则健康检查接口也会要求 JWT 认证。

6. **APP_GUARD 与 APP_INTERCEPTOR 顺序**：NestJS 中 Guard 在 Interceptor 之前执行，全局注册顺序不影响执行顺序。

7. **bcrypt 异步操作**：始终使用 `bcrypt.compare()`（异步），禁止使用 `bcrypt.compareSync()`，避免阻塞事件循环。
