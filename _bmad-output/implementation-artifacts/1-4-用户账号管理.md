# Story 1.4: 用户账号管理

Status: ready-for-dev

## Story

As a 系统管理员,
I want 在设置页面创建、查看、编辑、停用用户账号,
so that 能够灵活管理系统用户，控制账号访问权限.

## Acceptance Criteria

1. **用户列表页面**
   - 系统管理员可查看所有用户列表（用户名、姓名、账号状态）
   - 支持按用户名/姓名搜索
   - 支持按账号状态筛选（活跃/停用）
   - 支持分页（默认 20 条/页）

2. **用户详情页面**
   - 系统管理员可查看用户详细信息（用户名、姓名、账号状态、创建时间、更新时间）
   - 显示操作审计信息（created_by / updated_by）

3. **创建用户**
   - 系统管理员可创建新用户（用户名、姓名、初始密码）
   - 用户名必须唯一
   - 新用户默认状态为活跃（ACTIVE）
   - 创建操作自动记录 created_by 字段

4. **编辑用户**
   - 系统管理员可编辑用户信息（姓名、密码）
   - 编辑操作自动记录 updated_by 字段

5. **停用用户**
   - 系统管理员可停用用户账号
   - 停用账号立即失效，持有的 Token 在下次请求时被拒绝
   - 停用操作需二次确认

6. **权限控制**
   - 仅系统管理员可访问用户管理功能
   - 用户管理 API 需 JWT 认证

## Dev Notes

### 架构模式和约束

**后端架构（来自 architecture.md）：**
- User 实体继承 BaseEntity（含 created_at / updated_at / created_by / updated_by）
- 用户表字段：username（唯一）、name、password（@Exclude）、status（ENUM: active/inactive）
- 所有实体字段标注 @Expose()，@Column 显式指定 snake_case 列名
- 停用账号检查：在 JwtStrategy.validate() 中验证用户 status，若为 inactive 则拒绝

**前端架构（来自 UX 设计）：**
- 用户管理页面：列表页（搜索/筛选/分页）+ 详情页 + 表单页（新建/编辑）

### 项目结构注记

**命名规范：**
- 数据库：snake_case（表名复数、列名、外键 {实体}_id）
- API 端点：kebab-case（`/api/v1/users`、`/api/v1/users/:id`）
- 代码：camelCase（变量/函数）、PascalCase（类/接口/类型）
- 文件：kebab-case（后端）、PascalCase（React 组件）

**通信模式：**
- 响应格式：`{ success, data, message, code }`
- 分页：Offset 分页（page + pageSize，默认 20）
- 错误码：HTTP 状态码 + 业务 code（如 ACCOUNT_DISABLED）

**执行规范：**
- 所有实体继承 BaseEntity（4 个审计字段）
- 所有列表接口支持 page + pageSize 分页
- 禁止在组件内直接调用 axios，必须通过 API 层
- 所有枚举定义在 packages/shared（前后端共享）
