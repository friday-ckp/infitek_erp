# Story 3.6: 产品资料库管理

Status: done

## Story

As a 产品专员,
I want 在产品资料库中创建和管理资料记录（彩页、规格说明书、证书、图片、视频等），上传文件，记录归属类型和国家/地区，
So that 所有产品文档有统一的存储和检索入口，不再散落在各人电脑里。

---

## Acceptance Criteria

**Given** 产品专员进入产品资料库列表页
**When** 页面加载完成
**Then** 展示资料列表（ProTable），含列：资料名称、资料类型、归属类型、所属产品/分类、国家/地区、上传时间
**And** 支持按资料类型/归属类型筛选、关键字搜索、分页

**Given** 产品专员点击"新建资料"
**When** 进入资料创建表单
**Then** 表单包含：资料名称（必填）、资料类型（必填，枚举下拉）、资料内容（富文本，选填）、资料文件（必填，OSS 上传）、归属类型（必填，枚举下拉）、国家/地区（选填，单选）、所属产品/分类（根据归属类型联动显示）
**And** 文件上传至阿里云 OSS（使用已有 FilesService）

**Given** 产品专员选择归属类型
**When** 归属类型 = "通用归属"
**Then** 不显示所属产品/分类字段

**Given** 产品专员选择归属类型
**When** 归属类型 = "产品一级分类归属"
**Then** 显示"所属产品一级分类"字段（TreeSelect，仅显示 level=1 节点）

**Given** 产品专员选择归属类型
**When** 归属类型 = "产品二级分类归属"
**Then** 显示"所属产品二级分类"字段（TreeSelect，仅显示 level=2 节点）

**Given** 产品专员选择归属类型
**When** 归属类型 = "产品三级分类归属"
**Then** 显示"所属产品三级分类"字段（TreeSelect，仅显示 level=3 节点）

**Given** 产品专员选择归属类型
**When** 归属类型 = "产品归属"
**Then** 显示"所属产品"字段（Select，搜索 SPU）

**Given** 任意用户在资料详情页
**When** 点击"下载文件"
**Then** 调用 FilesService.getSignedUrl 获取签名 URL，浏览器打开下载

**Given** 产品专员删除资料记录
**When** 系统弹出 Modal 二次确认（"删除后文件将从云存储永久删除，无法恢复"）
**Then** 确认后执行 OSS 物理删除（FilesService.delete）+ 数据库记录删除

## Dev Agent Context

### 关键架构决策

**FilesService 已是全局模块（isGlobal: true），无需 import 即可注入：**
```typescript
constructor(
  private readonly repo: ProductDocumentsRepository,
  private readonly filesService: FilesService,
  private readonly spusService: SpusService,
  private readonly categoriesService: ProductCategoriesService,
  private readonly countriesService: CountriesService,
) {}
```

**文件上传策略（两步式）：**
1. 前端先调用 `POST /api/files/upload?folder=product-documents` 上传文件，获得 `{ key, filename, size }`
2. 前端将 `fileKey` 和 `fileName` 作为普通 JSON 字段提交到 `POST /api/product-documents`
3. Controller 接收标准 JSON body，不需要 FileInterceptor

**归属类型决定哪个分类/产品字段有值：**
- `general`（通用归属）→ 所有分类/产品字段为 null
- `category_l1`（产品一级分类归属）→ `categoryLevel1Id` 有值，其余为 null
- `category_l2`（产品二级分类归属）→ `categoryLevel2Id` 有值，其余为 null
- `category_l3`（产品三级分类归属）→ `categoryLevel3Id` 有值，其余为 null
- `product`（产品归属）→ `spuId` 有值，其余为 null

**资料内容（content）为富文本，存 longtext：**
前端使用 `ProFormTextArea`（rows=6）模拟富文本，后端存 longtext。

#### Repository

```typescript
async findAll(query: QueryProductDocumentDto) {
  const { keyword, page = 1, pageSize = 20, documentType, attributionType, spuId, countryId } = query;
  const qb = this.repo.createQueryBuilder('d');

  if (keyword) {
    qb.where('(d.document_name LIKE :kw)', { kw: `%${keyword}%` });
  }
  if (documentType) qb.andWhere('d.document_type = :documentType', { documentType });
  if (attributionType) qb.andWhere('d.attribution_type = :attributionType', { attributionType });
  if (spuId !== undefined) qb.andWhere('d.spu_id = :spuId', { spuId });
  if (countryId !== undefined) qb.andWhere('d.country_id = :countryId', { countryId });

  const [list, total] = await qb
    .orderBy('d.created_at', 'DESC')
    .skip((page - 1) * pageSize)
    .take(pageSize)
    .getManyAndCount();
  return { list, total, page, pageSize, totalPages: Math.ceil(total / pageSize) };
}
```

#### Service

```typescript
async create(dto: CreateProductDocumentDto, operator?: string) {
  // 可选校验：spuId、categoryId、countryId 存在性
  if (dto.spuId) {
    const spu = await this.spusService.findById(dto.spuId);
    if (!spu) throw new NotFoundException(`SPU ${dto.spuId} 不存在`);
  }
  const entity = await this.repo.create({
    documentName: dto.documentName,
    documentType: dto.documentType,
    content: dto.content ?? null,
    attributionType: dto.attributionType,
    countryId: dto.countryId ?? null,
    categoryLevel1Id: dto.categoryLevel1Id ?? null,
    categoryLevel2Id: dto.categoryLevel2Id ?? null,
    categoryLevel3Id: dto.categoryLevel3Id ?? null,
    spuId: dto.spuId ?? null,
    fileKey: dto.fileKey ?? null,
    fileName: dto.fileName ?? null,
    createdBy: operator,
    updatedBy: operator,
  });
  return entity;
}

async remove(id: number) {
  const existing = await this.repo.findById(id);
  if (!existing) throw new NotFoundException('资料不存在');
  if (existing.fileKey) await this.filesService.delete(existing.fileKey);
  await this.repo.delete(id);
}
```

### 避坑指南（必读）

1. **FilesService 是 @Global()，直接注入，ProductDocumentsModule 无需 import FilesModule**

2. **Entity 无软删除：** BaseEntity 无 `deletedAt`，`repo.delete(id)` 直接物理删除

3. **CountriesModule 需确认是否 export CountriesService：** 检查 `countries.module.ts` 的 `exports`，如未导出则添加

4. **归属类型联动：** 前端切换归属类型时必须清空其他关联字段，避免提交时带上无效数据

5. **分类 TreeSelect 数据：** 复用 SPU 表单中的 `buildTreeData` 方法，按 level 过滤节点

6. **国家/地区单选：** 调用 `GET /api/countries?pageSize=300` 获取全量列表，转为 Select options

7. **content 富文本：** 当前用 ProFormTextArea 实现，后续可替换为富文本编辑器，数据库已用 longtext 预留空间

8. **Migration 时间戳：** 使用 `20260424000100`，避免与已有 migration 冲突
